## Community practice: managing skills-only repos

**Example repos (verified by fetch unless noted):**
- **anthropics/skills** (github.com/anthropics/skills) — official. Organized by category (`skills/` creative, dev, enterprise, document-skills; `spec/` for the Agent Skills spec; `template/`). Each skill = folder + SKILL.md (only `name`/`description` required in frontmatter). **No formal test/eval process** — README explicitly disclaims: "provided for demonstration... always test thoroughly yourself." No per-skill versioning. **Is** published as a Claude Code plugin marketplace (`/plugin marketplace add anthropics/skills`, then `/plugin install <set>@anthropic-agent-skills`).
- **obra/superpowers** (github.com/obra/superpowers) — community, most eval-mature example found. Skills in `/skills` by category (testing, debugging, collaboration, meta). Uses semantic versioning (v6.1.1, `.version-bump.json`, real releases). Explicitly restricts new-skill contributions and requires cross-agent compatibility (Claude/Cursor/Copilot CLI/etc.) — a real cited maintenance burden.
- **alirezarezvani/claude-skills**, **Jeffallan/claude-skills**, **hesreallyhim/awesome-claude-code**, **travisvn/awesome-claude-skills**, **ComposioHQ/awesome-claude-skills** — large curated/aggregator collections (300+ skills), search-snippet-level only, not fetched — these look like flat folder dumps with no visible eval process, just README-level curation.

**Eval-gated publishing — found ONE real example, verified:**
`prime-radiant-inc/superpowers-evals`, wired into obra/superpowers via `evals/` (cloned in, per CONTRIBUTING). Called "drill eval harness" / codename "Quorum": drives real tmux sessions of Claude Code/Codex/Gemini/Kimi through a QA agent, grades **workflow compliance** against YAML scenario files (`evals/scenarios/*.yaml`) plus deterministic post-checks. Scenarios take 3–30+ min each (real LLM sessions, not mocked). Docs say skill-behavior changes should run through this before landing — this is the closest thing to "eval-gated skill publishing" I found in the wild, though I could not verify it's a *hard-blocking* CI gate (no CI config fetched) vs. a recommended pre-merge step.

**Anthropic's own answer to this problem — skill-creator (test/measure/refine), verified by fetch:**
claude.com/blog/improving-skill-creator-test-measure-and-refine-agent-skills — lets a non-engineer author write test prompts + "what good looks like" as evals; skill-creator runs them and reports pass/fail. Has a **benchmark mode** tracking pass rate, elapsed time, token usage across iterations for A/B comparison, and explicitly recommends re-running evals against new models as "an early signal when something shifts." Notably: **this is opt-in, not gated** — "your evals and results stay with you... plug them into a CI system [if you want]." So Anthropic has built the tooling for eval-gating but does not itself enforce it as a publishing requirement.

**Pain points found (with sources):**
- **Skill drift vs. source-of-truth**: HN thread on an email-marketing skill (found via hn.algolia.com search) — "the skill and the Bible can drift... needs to be manually kept in sync," wanting an MCP-based auto-propagation fix.
- **Context budget pressure**: same thread — "the full 55K-word guide is too big for a context window," forcing lossy condensation into skills.
- **Silent breakage / non-obvious debugging**: buildtolaunch.substack.com/p/claude-skills-not-working-fix — skills silently misfire from broken paths, orphaned files, command-name collisions, with "none of these surface as errors."
- **Academic framing of the exact problem**: arXiv paper "Skill Drift Is Contract Violation: Proactive Maintenance for LLM Agent Skill Libraries" (arxiv.org/pdf/2605.10990) formalizes skill decay as environment-contract violation as APIs/configs evolve underneath a skill — I did not fetch full text, snippet-only.
- Multiple secondary blog posts (dev.to, claudeai.dev) converge on the same folk remedy: "treat skills like code — monthly audits, per-step constraints, external feedback over self-revision" — all snippet-level, not deep-verified.

**Bottom line:** No repo enforces eval-gating as a hard requirement today. superpowers/superpowers-evals is the most credible real-world instance of behavioral-eval infrastructure attached to a skill library; Anthropic's skill-creator provides the tooling for teams to self-gate but stops short of mandating it. This is a real, currently-unfilled gap — consistent with a "Skillmaker Studio" product thesis.
