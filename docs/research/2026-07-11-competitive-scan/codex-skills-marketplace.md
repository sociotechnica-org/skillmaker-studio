**1. Skills — YES, confirmed.** Codex CLI ships a native "skills" concept: a directory with required `SKILL.md` (name+description frontmatter) plus optional `scripts/`, `references/`, `assets/`, `agents/openai.yaml`. Discovered from repo-level `.agents/skills` (walked up to repo root), user-level `$HOME/.agents/skills`, admin `/etc/codex/skills`, and built-in system skills. Explicitly built on the **open agent skills standard** (agentskills.io / github.com/openai/skills), the same SKILL.md spec Claude Code uses — so skills are cross-compatible in principle. **3. Marketplaces registered via `.agents/plugins/marketplace.json` (repo or `~/.agents/plugins/`), sourced from git repos (full repo or subdir, with ref/sha pinning), local dirs, or npm packages. CLI: `codex plugin marketplace add owner/repo`, `list`, `upgrade`, `remove`; `/plugins` in the TUI opens a searchable plugin browser (install/uninstall/toggle). OpenAI runs a "curated marketplace that powers the official Plugins Directory," with a formal submission portal (`/codex/submit-plugins`) — self-serve publishing not yet fully open per some secondary sources, but the marketplace mechanics themselves are official and documented, not a hack. Confirmed: https://learn.chatgpt.com/docs/plugins and https://learn.chatgpt.com/docs/build-plugins (redirected from developers.openai.com/codex/plugins, /codex/build-plugins).

**4. AGENTS.md — confirmed NOT a marketplace/plugin system.** It's a single-file "README for agents" convention (build/test/style context), hierarchical (AGENTS.override.md > AGENTS.md, walked root→cwd), multi-tool (Codex, Amp, Jules, Cursor, Factory), now stewarded by the Agentic AI Foundation / Linux Foundation. Confirmed: https://github.com/openai/codex/blob/main/docs/agents_md.md, https://agents.md/ (search-snippet level for PRPM claim, not fetched).
